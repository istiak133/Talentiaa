## 1) Inconsistency Fixes (Resolved With Justification)

### 1. Refresh token 7d vs Remember me 30d
Final rule:
- Access token: 15 minutes
- Refresh token (default): 7 days
- Remember me on হলে refresh token TTL: 30 days
- Absolute max session lifetime: 30 days
Justification:
- Security + usability দুইটাই balance হয়
- Recruiter বারবার login করতে হয় না, আবার indefinite sessionও থাকে না

### 2. Job Type vs Remote/Hybrid/Onsite mix
Final rule:
- jobType: Full-time, Part-time, Contract, Internship
- workplaceType: Onsite, Hybrid, Remote
Justification:
- Industry standard data model
- Filtering accurate হয়

### 3. Hidden Pool + Auto rejection email conflict
Final rule:
- Hidden Pool এ গেলে rejection email যাবে না
- শুধু "Under review" বা "Application received" status থাকবে
- Final rejection action trigger হলে rejection email যাবে
Justification:
- পরে recover করলে trust break হয় না
- Candidate experience clean থাকে

### 4. Identity masking vs recruiter email search
Final rule:
- Recruiter search করতে পারবে: aliasId, skills, score, stage, experience
- Recruiter email/phone search করতে পারবে না
- Admin প্রয়োজন হলে full PII search করতে পারবে
Justification:
- Data privacy policy enforce হয়
- Masking শুধু UI-level না, API-level হয়

### 5. Recruiter invite-only + Google OAuth undefined
Final rule:
- Candidate: email/password বা Google OAuth
- Recruiter: invite-only, invited email দিয়েই OAuth allow
- Admin: seeded account + controlled creation
Justification:
- Unauthorized recruiter sign-up বন্ধ থাকে
- OAuth UX benefits পাওয়া যায়

### 6. Review vs Shortlisted naming mismatch
Final rule:
- Canonical stage IDs: REVIEW, INTERVIEW, OFFER, HIRED, REJECTED
- UI label হিসেবে "Shortlisted" optional alias, internal ID change হবে না
Justification:
- API, analytics, automation stable থাকে

### 7. Transactional email এ unsubscribe
Final rule:
- Transactional emails: no unsubscribe link
- Optional emails (daily digest, product tips): unsubscribe link mandatory
Justification:
- Compliance ঠিক থাকে
- Critical workflow mails accidentally বন্ধ হয় না

### 8. Custom scoring weights vs comparability
Final rule:
- Job publish হওয়ার আগে recruiter weights set করতে পারবে
- First application আসার পর scoringConfig lock হবে
- Change দরকার হলে scoringVersion++ (v2) with explicit recalc action
Justification:
- Historical scores comparable থাকে
- Auditability improve হয়

### 9. Real-time update conflict handling missing
Final rule:
- Application document এ version field থাকবে
- Stage move API optimistic concurrency check করবে
- Conflict হলে 409 + refresh prompt
Justification:
- দুই recruiter একসাথে move করলে data overwrite হয় না

### 10. OpenAI-only parsing vs free-first requirement
Final rule:
- Provider abstraction mandatory
- Primary: Gemini Flash free tier বা Groq free tier
- Fallback: rule-based parser + manual correction
Justification:
- Free-first requirement maintain হয়
- API outage হলেও workflow বন্ধ হয় না

## 2) Best Tech Stack (This Scope-এর জন্য)

### Frontend
- React + Vite + TypeScript
- Tailwind CSS + shadcn/ui
- React Router
- TanStack Query
- React Hook Form + Zod
- TanStack Table
- dnd-kit (Kanban)
- Recharts (Analytics)

Why:
- Fast dev speed + enterprise-grade UI + predictable state management

### Backend (MVC)
- Node.js + Express + TypeScript
- Mongoose (MongoDB)
- JWT + HttpOnly cookies
- Passport (Google OAuth)
- BullMQ (notifications/async jobs)
- Socket.IO (live updates in pipeline)
- Pino logger

Why:
- তোমার required features-এর জন্য mature এবং solo-dev friendly

### Database/Infra
- MongoDB Atlas (primary DB)
- Redis (queue + realtime assist)
- Cloudinary বা S3-compatible storage (CV files)

Why:
- একটাই main DB দিয়ে কাজ হবে, supporting services শুধু operational need মেটাবে

### AI/Parsing
- pdf-parse + mammoth (text extraction)
- Gemini Flash free tier or Groq free tier (structured parsing)
- Rule-based fallback parser

Why:
- Free-first + reliability

### Email/Notification
- Brevo free tier বা Resend free tier
- React Email templates
- BullMQ queue + retry + dead-letter policy

### Security
- Helmet
- CORS allowlist
- Rate limiting
- Brute-force guard
- CSRF protection
- Argon2 password hashing
- Audit logs for critical actions

## 3) Simple But Practical DB Design (Role-based merge করা final)

1. users
- identity, role, auth status, verify status

2. recruiter_invites
- invitedEmail, invitedBy, tokenHash, expiresAt, acceptedAt

3. jobs
- job details, jobType, workplaceType, scoringConfig, threshold, status

4. candidate_profiles
- parsed profile + manual edits

5. resumes
- file metadata, parsing status, parse result, provider info

6. applications
- jobId, candidateId, aliasId, currentStage, stageHistory, scoreSummary, hiddenPoolFlag, version

7. notifications
- in-app events + email dispatch state

8. email_templates
- customizable optional templates

9. activity_logs
- auth events, stage move, admin actions, scoring recalcs

10. analytics_events (বা aggregation-first approach)
- dashboard metrics build করতে event trail

Note:
- Over-optimization করো না
- Phase-1 এ কম index, Phase-2 এ query profiling দেখে add করো
