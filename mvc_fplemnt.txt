## ✅ Consistency Fixes Applied Globally
1. Refresh token conflict resolved:
- Access token: 15 minutes
- Refresh token: 7 days
- Remember me on: 30 days refresh token
- Absolute session max: 30 days

2. Job fields standardized:
- jobType: Full-time, Part-time, Contract, Internship
- workplaceType: Onsite, Hybrid, Remote

3. Hidden Pool behavior fixed:
- Hidden Pool এ গেলে auto rejection email যাবে না
- Final rejection action এ rejection mail যাবে

4. Privacy rule enforced:
- Recruiter candidate email/phone search করতে পারবে না
- Recruiter search করবে alias/skills/score/stage/experience দিয়ে

5. Recruiter OAuth rule fixed:
- শুধুমাত্র invited recruiter email এর জন্য OAuth allowed

6. Stage naming fixed:
- Internal canonical: REVIEW, INTERVIEW, OFFER, HIRED, REJECTED
- UI তে চাইলে REVIEW label কে "Shortlisted" দেখানো যাবে

7. Email unsubscribe fixed:
- Transactional emails: no unsubscribe
- Optional emails (digest/promotional): unsubscribe mandatory

8. Scoring comparability fixed:
- First application আসার পর scoringConfig lock
- Change করলে scoringVersion++ + explicit recalc

9. Realtime conflict fixed:
- Optimistic concurrency with version check
- Conflict হলে API 409 ফেরত দেবে

10. Free-first AI parsing fixed:
- Provider abstraction
- Primary free provider (Gemini Flash/Groq free)
- Fallback rule-based + manual correction

11. Job draft model simplified:
- আলাদা JobDraft collection থাকবে না
- Job model এই status="draft" ব্যবহার হবে

12. Recruiter invite expiry fixed:
- Invite validity: 72 hours
- status: pending | accepted | expired
- usedAt tracked থাকবে

```

```

## 2. SMART JOB POSTING

```
Dashboard -> "Post New Job"

- Job title লিখে

Step 2: Job Details
- Department
- jobType (Full-time/Part-time/Contract/Internship)
- workplaceType (Onsite/Hybrid/Remote)
- Location
- Salary range + show/hide toggle
- Experience level (Junior/Mid/Senior/Lead)

- Required Skills tag add করে

## 6. AI MATCH SCORING & RANKING

### কীভাবে score হয় (Example)
```
Job Requirements:
- Skills: React, Node.js, MongoDB, TypeScript
- Experience: 3+ years
- Education: BSc CS/CSE related

Candidate Profile:
- Skills: React, Node.js, MongoDB, Python
- Experience: 2.5 years
- Education: BSc in CSE

Skill score: 75
Experience score: 83
Education score: 100

Weighted score =
Skills(50%) + Experience(35%) + Education(15%)
= 82%
```

### Candidate এর অভিজ্ঞতা:
```
নিজ dashboard এ application overall score দেখতে পারে (e.g. 82%)
-> score breakdown, missing skills, weighted details দেখতে পায় না
```

### Recruiter এর অভিজ্ঞতা:
```
Candidate ranked list:
- score high থেকে low
- score breakdown expandable
- threshold set (e.g. 70%)
- threshold এর নিচে Hidden Pool
- recover button দিয়ে pipeline এ ফেরত আনা যায়
```

### Admin এর অভিজ্ঞতা:
```
Scoring audit এবং version changes দেখে
```

### Industry Standard Implementation
- Explainable score breakdown
- Scoring weights configurable only before first application
- First application পর scoring config lock
- New config হলে scoringVersion++
- Hidden Pool no rejection mail
- Candidate visibility policy:
  - Candidate: overall score only
  - Recruiter/Admin: full score breakdown

### MVC Implementation Steps

3. ActivityLog

#### Controllers

#### Services

#### Routes

2. POST /system/applications/:applicationId/score
3. GET /recruiter/jobs/:jobId/candidates/ranked
4. POST /recruiter/applications/:applicationId/recover-hidden

### End-to-End Flow
```
Application created
-> scoring run
-> score + explanation store
-> threshold check
-> pipeline বা hidden pool route
